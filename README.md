# pi-coding-agent

I designed a pi-coding-agent project to convert the jsonl files produced by pi-coding-agent into something useful for a Human in the Loop to read.
The final .csv file output by the code designed by this agent just looks at the prompt and responses generated for each agent and how many tokens were consumed in the process.
There's a lot of rows with neither prompts or responses.  These are thinking steps so the tokens consumed are still published in the .csv file.

This repo contains 
* Framework files behind my pi-coding-agent (AGENTS.md, orchestrator-policy.yaml & models.json)
* Source files for the agent to ingest (example-input.jsonl & example-output.csv)
* A slightly amended (for input filepath/name) version of the .py code generated by the agent
